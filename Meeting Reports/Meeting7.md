* Branching Conventions

We decided to use a clear and consistent format for our Git branches to have a better collaboration and organization.  

| Branch Type      | Prefix                   | Use case |  Example |
|:----------------:|:------------------------:|:--------:|:--------:|
| Feature          | `feature/` or `feat/`    | for developing new features | feature/login-system |
| Bugfix           | `bugfix/`                |  to fix bugs in the code |  bugfix/header-styling |
| Hotfix           | `hotfix/`                | to fix critical bugs in the production environment | hotfix/critical-security-issue |
| Release          | `release/`               | for a new production release | release/v1.0.1 | 
| Documentation    | `docs/`                  | to write, update, or fix documentation |  docs/api-endpoints |
| Test             | `test/`                  | for experimenting outside of an issue/ticket | test/dark-mode-colours |
