### **Discussion :**
During this meeting, 3 main topics were discussed: branching convention, viusal identity of our website, the personality quiz.  

When reviewing the meeting report6, we first discussed the branch naming convention that will be used for the project. We decided to use a clear and consistent format for our Git branches to have a better collaboration and organization.  

* Branching Conventions

| Branch Type      | Prefix                   | Use case |  Example |
|:----------------:|:------------------------:|:--------:|:--------:|
| Feature          | `feature/` or `feat/`    | These branches are used for developing new features | feature/login-system |
| Bugfix           | `bugfix/`                | These branches are used to fix bugs in the code |  bugfix/header-styling |
| Hotfix           | `hotfix/`                | These branches are made directly from the production branch to fix critical bugs in the production environment | hotfix/critical-security-issue |
| Release          | `release/`               | These branches are used to prepare for a new production release | release/v1.0.1 | 
| Documentation    | `docs/`                  | These branches are used to write, update, or fix documentation |  docs/api-endpoints |
| Test             | `test/`                  | These branches are for experimenting outside of an issue/ticket | test/dark-mode-colours |

* Personality quiz

Another important point we talked over was the personality quiz. As the quiz's name will be *"The IT match"*, we thought about making it a swipe-based quiz, with questions similar to “would you rather” questions where the user chooses between two options by swiping left (for No)  or right (for Yes).  
We also reviewed some example questions that we had prepared for the quiz. They are based on different personality dimensions such as:
- Logical vs Creative.
- Solo vs Team.
- Theoretical vs Practical.
- Low-level vs High-level.
- Math-oriented vs Intuition-driven.
  
Some example questions could be :
- When organizing a party, do you prefer making a detailed checklist or brainstorming wild ideas?
- When facing a tricky problem, do you follow a proven method or experiment until something works?
- In a project, do you prefer working alone or collaborating with others?
- When learning something new, do you prefer a solo tutorial or a hands-on workshop?
- When studying, do you prefer understanding why something works or learning directly how to use it?
